# 编译路径解析优化计划

用户报告在解析 FPC 编译输出时，无法正确获取 `Posix_SysMman.pas` 的真实位置。经过分析，原因在于：
1. **正则表达式限制**：现有的匹配 "Compiling" 文本的正则要求其位于行首，但有些 FPC 配置会带有数字前缀（如 `(3104) `），导致上下文文件 `this.currentFile` 未被更新。
2. **相对路径解析不足**：当错误发生在一个包含文件（如 `.inc`）中时，当前的逻辑主要依赖工作区搜索，而不考虑当前正在编译的文件所在目录。

## 任务目标
- 修改 `BaseBuildTerminal` 类，使其能正确解析带有消息 ID 前缀的编译信息。
- 增强相对路径解析逻辑，优先检查当前编译文件所在的目录。

## 详细变更

### 1. 修改 `src/providers/baseBuildTerminal.ts`
- **更新 `compileMatch` 正则**：改为支持可选的消息 ID 前缀。
- **改进 `parseFpcStyleError` 中的路径补全**：
    - 如果识别出的文件名是相对路径，首先尝试在 `this.currentFile` 所在的目录下查找该文件。
    - 如果上述查找失败，再进行现有的工作区全局搜索。

## 验证计划
- 确认 `(3104) Compiling ...` 形式的输出能正确更新 `this.currentFile`。
- 确认包含文件（如 `SysMmanAPI.inc`）即使在工作区外（但在主文件同目录时）能被正确定位。
